# 医疗器械借用、消毒与报损追踪系统

## 项目用途

本系统是一款面向医院的医疗器械全生命周期管理系统，覆盖设备从入库、借用、消毒到报损的完整流程。系统旨在规范医疗器械的使用管理，确保高风险器械的消毒质量，防止交叉感染，提高设备利用率，同时满足医疗监管的可追溯要求。

### 核心功能

- **器械台账管理**：维护设备基本信息，包括序列号、风险等级、消毒要求、当前位置和可用状态
- **借用流程管理**：支持科室申请、库管审批、设备出库、使用归还的全流程管理
- **消毒队列管理**：设备归还后自动进入消毒队列，记录消毒方式、批次号和责任人
- **报损管理**：设备损坏或老化时提交报损申请，审批后设备标记为不可用
- **数据导出**：支持报损清单导出，包含借用人、消毒批次、报损原因和当前状态等信息
- **逾期提醒**：自动检测逾期未归还设备，顶部横幅提醒相关人员

## 技术栈

- **前端框架**：React 18 + TypeScript
- **构建工具**：Vite
- **状态管理**：Zustand（支持 localStorage 持久化）
- **路由**：React Router v7
- **样式**：Tailwind CSS
- **图标**：Lucide React

## 系统页面

### 1. 器械台账（首页）

**路径**：`/`

展示所有医疗器械的列表，支持：
- 按设备名称、序列号搜索
- 按风险等级（高/中/低）筛选
- 按状态（可用/借出/待消毒/消毒中/已报损）筛选
- 按所属科室筛选
- 新增设备
- 查看设备详情

设备信息包含：设备名称、序列号、风险等级、消毒要求、所属科室、当前位置、状态、入库日期。

### 2. 借用看板

**路径**：`/borrow`

管理设备借用全流程，支持：
- 查看所有借用记录
- 按状态筛选（待审批/待出库/借出中/已归还）
- 搜索设备或借用人
- 提交借用申请
- 审批借用申请
- 确认设备出库
- 设备归还（需填写归还科室）

**逾期提醒**：超过预计归还日期未归还的设备会在页面顶部显示红色警告横幅。

### 3. 消毒队列

**路径**：`/disinfect`

管理设备消毒流程，支持：
- 查看待消毒设备列表
- 查看消毒中设备列表
- 查看已完成消毒记录
- 开始消毒
- 完成消毒（需填写消毒方式、批次号、责任人）

**高风险提醒**：待消毒的高风险器械会有特别提醒，优先处理。

### 4. 报损导出

**路径**：`/damage`

管理设备报损和数据导出，包含两个标签页：

**报损管理**：
- 查看所有报损申请
- 提交报损申请
- 审批报损申请
- 显示最后借用人和最后消毒批次

**导出记录**：
- 按日期范围筛选报损记录
- 导出 CSV 格式报损清单
- 查看历史导出记录

导出内容包含：设备名称、序列号、借用人、消毒批次、报损原因、当前状态、报告人、报告日期。

## 业务规则与拦截机制

系统内置多项业务规则校验，确保流程合规：

### 1. 设备未消毒完成拦截

- 设备状态为「待消毒」或「消毒中」时，无法提交借用申请
- 设备有未完成的消毒记录时，审批环节会拦截

### 2. 同一设备重复借出拦截

- 设备已有进行中的借用单（待审批/已审批/已出库）时，无法再次申请
- 防止同一设备被多个科室同时预约

### 3. 报损设备继续分配拦截

- 设备已批准报损后，状态变为「已报损」
- 已报损设备无法被借用，借用申请环节直接拦截

### 4. 归还科室不一致拦截

- 设备归还时，归还科室必须与借出科室一致
- 防止设备在不同科室间流转而未登记

### 5. 消毒记录缺少责任人拦截

- 完成消毒时必须填写责任人
- 未填写责任人无法完成消毒流程
- 确保消毒质量可追溯

### 6. 高风险器械强制消毒

- 高风险器械必须有消毒记录才能借出
- 新增的高风险设备如无消毒记录，首次借用会被拦截
- 保障患者安全，防止交叉感染

## 验收路径

按照以下步骤进行系统验收：

### 验收一：器械台账管理

1. 访问首页「器械台账」
2. 点击「新增设备」，填写设备名称、序列号等信息，确认添加
3. 验证设备列表中出现新设备，状态为「可用」
4. 点击设备行或「详情」按钮，查看设备详情
5. 验证风险等级、所属科室、消毒要求等信息正确显示

### 验收二：借用流程

1. 进入「借用看板」页面
2. 点击「申请借用」，选择可用设备、填写借用人和预计归还日期，提交申请
3. 验证借用记录状态为「待审批」
4. 点击「审批」按钮，验证状态变为「待出库」
5. 点击「确认出库」，验证状态变为「借出中」，设备状态变为「借出」
6. 返回「器械台账」，验证对应设备状态已更新

### 验收三：归还与消毒

1. 在「借用看板」中找到借出中的设备，点击「归还」
2. 选择与借出科室**不同**的科室，验证提示「归还科室不一致」
3. 选择正确的归还科室，确认归还
4. 验证借用单状态变为「已归还」
5. 进入「消毒队列」，验证设备出现在「待消毒」列表
6. 点击「开始消毒」，验证状态变为「消毒中」
7. 点击「完成消毒」，不填写责任人直接提交，验证提示「消毒记录缺少责任人」
8. 填写完整信息后提交，验证状态变为「已完成」
9. 返回「器械台账」，验证设备状态恢复为「可用」

### 验收四：高风险器械强制消毒

1. 在「器械台账」中新增一台高风险设备（不要进行消毒）
2. 尝试申请借用该设备
3. 验证系统拦截，并提示「高风险器械强制消毒：该器械无消毒记录，不可借出」
4. 手动为该设备创建消毒记录（可通过先借出再归还的方式）
5. 完成消毒后，再次尝试借用，验证可以正常申请

### 验收五：逾期未归还提醒

1. 提交一笔借用申请，将预计归还日期设为过去的日期
2. 审批并出库该设备
3. 验证页面顶部出现红色「逾期未归还提醒」横幅
4. 验证「借用看板」中该设备行有红色高亮和「(逾期)」标记

### 验收六：报损流程

1. 进入「报损导出」页面
2. 点击「报损申请」，选择一台可用设备，填写报损人和原因
3. 验证报损记录状态为「待审批」
4. 点击「审批」，验证状态变为「已批准」
5. 返回「器械台账」，验证设备状态变为「已报损」
6. 尝试借用该报损设备，验证系统拦截

### 验收七：数据导出

1. 在「报损导出」页面切换到「导出记录」标签
2. 选择日期范围（可选），点击「导出报损清单」
3. 验证浏览器下载了 CSV 文件
4. 打开 CSV 文件，验证包含：设备名称、序列号、借用人、消毒批次、报损原因、当前状态等字段
5. 验证「导出记录」列表中出现了新的导出历史

### 验收八：数据持久化

1. 在系统中进行若干操作（新增设备、申请借用、完成消毒等）
2. 刷新浏览器页面
3. 验证所有数据保持不变，设备状态、消毒记录、报损单和导出历史均一致

## 关键数据流转

### 设备状态流转

```
可用 (available)
    │
    ▼ 提交借用申请
待审批借用中 ──► 可用 (审批拒绝)
    │
    ▼ 审批通过
已审批 (approved)
    │
    ▼ 库管出库
借出中 (borrowed / checked_out)
    │
    ▼ 设备归还
待消毒 (pending_disinfect)
    │
    ▼ 开始消毒
消毒中 (disinfecting / in_progress)
    │
    ▼ 完成消毒
可用 (available) ◄──┐
    │                │
    ▼ 报损申请       │ 复用
已报损 (damaged) ───┘
```

### 借用单状态流转

```
待审批 (pending)
    │
    ├─► 审批通过 ──► 已审批 (approved)
    │                      │
    │                      ▼ 库管出库
    └─► 审批拒绝        已出库 (checked_out)
                              │
                              ▼ 设备归还
                          已归还 (returned)
```

### 消毒记录状态流转

```
待消毒 (pending)  ── 设备归还时自动创建
    │
    ▼ 开始消毒
消毒中 (in_progress)
    │
    ▼ 完成消毒（需填写方式、批次、责任人）
已完成 (completed)  ── 设备恢复可用状态
```

### 报损单状态流转

```
待审批 (pending)
    │
    ├─► 审批通过 ──► 已批准 (approved) ──► 设备标记为已报损
    │
    └─► 审批拒绝 ──► （关闭申请，设备保持原状态）
```

## 数据持久化

系统使用 Zustand 的 `persist` 中间件，将所有状态数据存储在浏览器的 localStorage 中（key: `med-track-storage`）。刷新页面或关闭浏览器后重新打开，数据均保持一致。

持久化的数据包括：
- 设备列表及状态
- 借用记录
- 消毒记录
- 报损申请
- 导出历史

## 注意事项

1. 本系统为前端演示版本，数据存储在浏览器本地，适合演示和小规模使用
2. 生产环境需对接后端 API，实现多用户、权限管理等功能
3. 高风险器械的消毒要求应严格按照医院感控规范执行
4. 报损设备需由专人审核，确认后方可从可用库存中移除
5. 导出的 CSV 文件使用 UTF-8 编码，Excel 打开时请选择正确编码
